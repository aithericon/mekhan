# v0.1.0-alpha — first public drop

**Status: experimentation / early days.** APIs and schemas will change.

The Aithericon Platform is a Colored Petri-net workflow engine with a
real-time collaborative editor. This release is the first public source drop
— intended to develop in the open, not to be deployed.

## What's in the box

- **Engine** — Colored Petri-net execution core (event-sourced, hash-chained
  audit log), Rust SDK (`aithericon-sdk` + derive macros), CLI, simulator.
- **Executor** — Task/job runner with HDF5/NetCDF file metadata, Python IPC,
  Vault secret wrapping, Nomad + Slurm scheduler backends.
- **Service (`mekhan-service`)** — Axum control plane with Postgres-backed
  state, NATS messaging, Yjs collaborative editing.
- **App** — SvelteKit frontend (Svelte 5, xyflow, Yjs) with the template
  editor, instance view, and trigger catalog.

## Notable features landed

- SubWorkflow composition (inline + referenced) with engine-enforced borrow
  semantics on `<slug>.<field>` data refs.
- Control-flow primitives: Decision, Loop, Parallel — all proven via causality
  e2e tests.
- Catalog backfill + template-level single-active coalescing for triggers.
- Drag-into-container UX for Scope and Loop in the editor.
- Real-Nomad and real-Slurm scheduler e2e (`just dev scheduler-up` /
  `slurm-up`).

## Known limitations

- No stability guarantees on engine event schema, REST/OpenAPI surface, or DB
  migrations. Expect destructive migrations between releases.
- Cloud / enterprise components are not in this repo and load through stable
  interfaces only.
- Docs are uneven — architecture notes under `docs/` describe intent, not
  always the current state.
