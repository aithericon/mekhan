# Templates & Authoring

The authoring surface. A **workflow template** is a versioned `WorkflowGraph`
plus the compiled AIR JSON the engine runs. Templates live in a folder tree,
carry tags, and can be packaged into library packs.

# Schema — templates

| Table | Key columns | Purpose |
|-------|-------------|---------|
| `workflow_templates` | `id`, `name`, `version`, `is_latest`, `published`, `graph` (jsonb), `air_json` (jsonb), `interface_json` (jsonb), `workspace_id` → `workspaces.id`, `visibility`, `base_template_id`/`parent_id` (self FK), `pack_id` → `library_packs.id`, `template_kind`, `lifecycle_status`, `forked_from` (jsonb), `presentation` (jsonb), `metrics` (jsonb) | The core authoring object. `graph` is the editor model; `air_json` is the compiler output the engine deploys. Versioned: many rows per logical template, one `is_latest`. |
| `template_tags` | (`workspace_id` → `workspaces.id`, `base_template_id`, `tag`) | Free-form tags on the logical (base) template. |
| `webhook_slugs` | `id`, `workspace_id` → `workspaces.id`, `slug` (unique per ws), `template_id` → `workflow_templates.id`, `node_id` | Maps an external webhook URL slug to a trigger node on a template. |

# Schema — folders

| Table | Key columns | Purpose |
|-------|-------------|---------|
| `folders` | `id`, `workspace_id` → `workspaces.id`, `parent_id` (self FK), `slug`, `path`, `display_name`, `restricted` | Materialized-path folder tree, per workspace. |
| `template_folders` | (`base_template_id`, `folder_id` → `folders.id`, `workspace_id`), `moved_at`, `moved_by` | Places a logical template into a folder. |
| `pages` | `id`, `workspace_id` → `workspaces.id`, `title`, `attached_kind`, `attached_id`, `folder_id` → `folders.id` | Edra/Tiptap rich-text pages; free pages or notes attached to an object. The body is a [Yjs document](collaboration.md), not stored here. |

# Schema — library packs

| Table | Key columns | Purpose |
|-------|-------------|---------|
| `library_packs` | `id`, `workspace_id` → `workspaces.id`, `vendor`, `slug`, `version`, `name`, `origin`, `installed_by`, `installed_at` | A versioned bundle of reusable library nodes/templates; `workflow_templates.pack_id` points back here. |

# Schema — datacenter staging

| Table | Key columns | Purpose |
|-------|-------------|---------|
| `template_stagings` | `id`, `template_id` → `job_templates.id`, `template_version`, `datacenter_resource_id`, `status`, `remote_ref`, `last_error` | Tracks staging a **job template** version to a datacenter (note: FK is to [`job_templates`](fleet-and-compute.md), not `workflow_templates`). |

# Schema — template tests

| Table | Key columns | Purpose |
|-------|-------------|---------|
| `template_tests` | `id`, `template_id` → `workflow_templates.id`, `name`, `enabled`, `start_tokens` (jsonb), `human_answers` (jsonb), `assertions` (jsonb), `reference_scope` (jsonb), `last_run_passed` | A declarative test: seed tokens, scripted human answers, and assertions. |
| `template_test_runs` | `id`, `test_id` → `template_tests.id`, `instance_id`, `template_version`, `status`, `failure_detail` (jsonb), `final_scope` (jsonb), `duration_ms` | One execution of a test against a template version; the run is a real [test-mode instance](instances-and-execution.md). |

# Notes

- `visibility` (`private` / `workspace` / `public`) plus folder membership drives
  cross-workspace surfacing of the demo templates in the `demos` workspace.
- `air_json` is regenerated by the [service-side compiler](../../../CLAUDE.md) on
  every publish and POSTed to the engine to deploy the net.
